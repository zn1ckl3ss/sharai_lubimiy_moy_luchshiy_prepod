# DevSecOps для FastAPI: Полноценный CI/CD с безопасностью и мониторингом

Проект демонстрирует современный подход к разработке и эксплуатации веб-приложения на базе **FastAPI**. Реализован полный DevSecOps-конвейер с акцентом на безопасность кода, автоматизированное тестирование, инфраструктуру как код и систему наблюдения за работой сервиса.

## Основные технологии

**Основной стек:**
- **Backend**: Python 3.11 + FastAPI + Uvicorn
- **Контейнеризация**: Docker + Docker Compose
- **Оркестрация**: Kubernetes (манифесты)
- **CI/CD**: GitHub Actions (GitOps)
- **IaC**: Terraform
- **Конфигурация**: Ansible
- **Безопасность и качество**: Bandit (SAST), Ruff, Pylint
- **Тестирование**: Pytest + HTTPX
- **Мониторинг**: Prometheus + Grafana

**Целевая платформа**: Ubuntu 24.04 LTS (VPS 62yun)

---

## Как работает CI/CD

Каждый `git push` в основную ветку (`main`) запускает автоматизированный конвейер в GitHub Actions:

1. **Подготовка** — клонирование репозитория и настройка Python-окружения.
2. **Статический анализ безопасности** — Bandit сканирует код на уязвимости.
3. **Линтинг и форматирование** — Ruff (быстрая проверка + форматирование) и Pylint.
4. **Автотесты** — запуск тестов Pytest.
5. **Подготовка инфраструктуры** — Terraform через SSH настраивает сервер.
6. **Деплой** — Ansible устанавливает Docker, конфигурирует сервисы и запускает приложение.

## Ключевые технические решения

### Безопасность и качество кода
- **Bandit** выполняет глубокий AST-анализ и ищет типичные уязвимости Python-приложений.
- **Ruff** обеспечивает молниеносный линтинг и соблюдение PEP 8.
- **Pylint** проводит расширенный анализ архитектуры и сложности кода.
- Все проверки запускаются в чистом окружении GitHub Actions.

### Контейнеризация
Используется легковесный базовый образ `python:3.11-slim`. Готовый образ публикуется в Docker Hub (`soowhyy/sharai-app:latest`).

В приложение встроен сбор метрик через `prometheus-fastapi-instrumentator`. Метрики доступны по пути `/metrics`.

`docker-compose.yml` позволяет легко запускать весь стек локально (приложение + Prometheus + Grafana).

### Kubernetes-манифесты
- Два реплики приложения для отказоустойчивости.
- RollingUpdate-стратегия обновлений.
- Отдельный Namespace, ClusterIP Service и Ingress для внешнего доступа.

### Инфраструктура как код (Terraform)
Terraform использует `null_resource` + `remote-exec` для подготовки Ubuntu-сервера:
- Обновление системы.
- Установка необходимых пакетов.
- Безопасное подключение по SSH с помощью GitHub Secrets.

### Ansible-деплой
Плейбук выполняет идемпотентные операции:
- Установка Docker и Docker Compose.
- Создание директорий и конфигурационных файлов.
- Pull образа и запуск сервисов в detached-режиме.
